# Marvin Final Edit Checklist

## Content quality
- [ ] Clear thesis in first 2-3 paragraphs
- [ ] Target length is 1800-2500 words (deep-dive format)
- [ ] No fluff; each section adds practical value
- [ ] Claims are backed by sources from research note
- [ ] Balanced tone (confident, not alarmist)
- [ ] Includes at least one concrete case study or incident thread

## Readability
- [ ] Strong section titles
- [ ] Short paragraphs and scannable bullets
- [ ] Smooth transitions between sections
- [ ] Technical terms explained on first use

## SEO & metadata
- [ ] Title is specific and intent-driven
- [ ] Primary keyword appears naturally in intro + at least one heading
- [ ] Meta description ready (150-160 chars)
- [ ] Internal links to related posts included

## Publish readiness
- [ ] Front matter complete
- [ ] Filename follows `_posts/YYYY-MM-DD-slug.md`
- [ ] Final markdown validated locally
- [ ] `state.json` updated to `edited` or `published`
